# AI-OS v3 canon

## Core thesis

The system is not improved by adding more agents to the table.

The system improves when authority is split cleanly:

- one role routes
- one role owns strategy
- one role owns execution path and risk
- specialists produce domain work
- the human retains final authority

## Default operating model

Use daily mode unless the Founder explicitly turns on Zoom mode.

Daily mode exists to reduce token burn, reduce orchestration overhead, and keep small tasks out of committee dynamics.

## Zoom mode doctrine

Zoom mode is for complex, high-risk, ambiguous, or high-value work.

Its purpose is not to generate more words. Its purpose is to improve judgment quality by separating:

- proposal generation
- critique
- arbitration
- execution

## Dual-core philosophy

Dual-core does not simply mean "use two models".

The gain comes from structured adversarial pressure:

1. independent proposal generation
2. cross-critique
3. forced judgment
4. dissent preservation

This reduces echo-chamber behavior and surfaces disagreement earlier.

## Anti-mediocrity doctrine

Many AI review loops drift toward performative neutrality.

Reject that pattern.

If the system cannot choose a winner, it must say so explicitly and escalate.

Every merged decision should remain traceable to its originating side.

## Public deployment claim

This skill is self-contained:

- no external doctrine files are required
- no external templates are required
- one deployment prompt can attach the behavior layer

You may still adapt model names, tool names, or approval rules for a specific runtime.
